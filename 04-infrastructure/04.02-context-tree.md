# Context Tree (as Infrastructure)

This page describes Context Tree from the infrastructure perspective. For the full spec, see [02 How It Works](../02-how-it-works/NODE.md).

## Role in the stack

```
┌─────────────────────────────────────────────────┐
│                 AGENT TEAM                       │
│                                                  │
│  ┌──────────────┐  ┌──────────┐  ┌───────────┐  │
│  │ CONTEXT TREE │  │ Message  │  │Autonomous │  │
│  │ ◀── you are  │  │ System   │  │ Agents    │  │
│  │     here     │  │          │  │           │  │
│  └──────────────┘  └──────────┘  └───────────┘  │
│  ┌──────────┐  ┌──────────┐  ┌───────────────┐  │
│  │ Identity │  │ Database │  │   Workflow    │  │
│  └──────────┘  └──────────┘  └───────────────┘  │
└─────────────────────────────────────────────────┘
```

The Context Tree is the organizational memory. The other five pillars depend on it. Agents read the tree for context. The message system routes messages between node owners. Identity determines who can write to which nodes.

## How agents use it

```
1. Start at home node
2. Traverse up for broader context
3. Read NODE.md at each level
4. Follow soft links for related domains
5. Act with full team context
6. Write back: new decision → new node
7. Ask: does the tree need updating?
```

Retrieval happens three ways: reading NODE.md summaries, traversing the tree, and following soft links. A global semantic search layer can supplement traversal.

## The DRY principle

Information should have a single source of truth. Agents read the tree. They do not copy context between sessions. They reference nodes.

If the same decision is written in two places, one will become stale. One node. One truth.

## Bootstrapping

A new tree can be bootstrapped from:

- An existing codebase (mine commit history for design decisions)
- A document corpus (extract decisions from meeting notes, specs)
- From scratch (see [03.01 Quick Start](../03-getting-started/03.01-quick-start.md))

The tree grows with the organization. Start small. Draft first, restructure later.
