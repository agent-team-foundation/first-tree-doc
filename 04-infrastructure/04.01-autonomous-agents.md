# Autonomous Agents

An agent in an agent-centric organization is not a one-shot tool. It is a persistent participant.

```
┌─────────────────────────────────────────────────┐
│            AUTONOMOUS AGENT                      │
│                                                  │
│  Always online        Not just invoked by humans │
│  Has an inbox         Receives messages, events  │
│  Owns nodes           Approves changes to them   │
│  Has a home node      Starts there, builds       │
│                       context by traversal       │
│  Can spawn            Sub-agents for concurrent  │
│                       tasks                      │
│  Isolated runtime     Own environment, own state │
└─────────────────────────────────────────────────┘
```

## Requirements

**Always online.** The agent runs continuously. It can receive messages, respond to events, and act proactively.

**Isolated runtime.** Each agent has its own execution environment. Reproducible and isolated from other agents.

**Spawning.** An agent can spawn sub-agents or dispatch instances to handle concurrent tasks without blocking its main thread.

**Context Tree access.** Each agent has a home node. It loads context by traversing the tree from that node.

**Message system participation.** The agent has an inbox. It can send messages to other agents or humans and be awakened by incoming messages or external events.

**Node ownership.** An agent owns one or more nodes. Changes to those nodes require its approval. All owners must approve (AND logic).

## Interoperability

Any agent that meets these requirements can participate. Kael, Claude Code, open source agents. The infrastructure is agent-agnostic.

The standard does not require a specific agent framework. It requires specific capabilities: persistence, ownership, messaging, tree access.
