# Database

The Context Tree records decisions. Agents also need a place for operational data: state that changes frequently, is queried programmatically, and does not belong in a markdown file.

## Three layers

```
┌─────────────────────────────────────────────────┐
│                                                  │
│  Structured Data                                 │
│  Tables, records, relationships.                 │
│  Inventory, financials, user data, task state.   │
│  Agents query with SQL.                          │
│                                                  │
├──────────────────────────────────────────────────┤
│                                                  │
│  Data Lake                                       │
│  Raw and semi-structured.                        │
│  Meeting recordings, transcripts, logs,          │
│  conversation histories, uploaded files.          │
│  S3 bucket with a defined schema.                │
│                                                  │
├──────────────────────────────────────────────────┤
│                                                  │
│  Data Warehouse                                  │
│  Curated subsets of the lake.                    │
│  Optimized for analysis.                         │
│  Built incrementally as needs become clear.      │
│                                                  │
└─────────────────────────────────────────────────┘
```

## Why it is infrastructure

With a database, workflow engine, message system, and the ability to write code, an agent team can replace many point SaaS tools. The database is what transforms agents from conversational assistants into operational participants.

## Access model

- Agents read and write within their authorized scope.
- Data agents (specialized for analysis and data engineering) operate on lake and warehouse layers.
- Integrates with existing platforms (Snowflake, Databricks, cloud-native stores). Does not replace them.
