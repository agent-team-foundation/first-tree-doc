# Contributing

Context Tree is open source. Contributions are welcome.

## How to contribute

1. Read the doc. Understand the standard.
2. Pick an area: spec, guides, examples, CLI, or the doc itself.
3. Open an issue or PR on the relevant repo.
4. A node owner will review.

## Repos

- [context-tree-doc](https://github.com/agent-team-foundation/context-tree-doc) — this documentation
- [first-tree](https://github.com/agent-team-foundation/first-tree) — the original Context Tree, defines the standard
- [context-tree-blog](https://github.com/agent-team-foundation/context-tree-blog) — blog posts
- [context-tree-website](https://github.com/agent-team-foundation/context-tree-website) — the website

## What makes a good contribution

- Fix something that is wrong or unclear in the doc.
- Add an example tree that shows the standard in practice.
- Propose a new feature or pattern via a GitHub issue.
- Write a guide for a specific use case or tool integration.

## What to avoid

- Do not add execution details to the tree. The tree is the decision layer.
- Do not restructure without discussion. Open an issue first.
- Do not add nodes without owners. Every node needs an owner.
