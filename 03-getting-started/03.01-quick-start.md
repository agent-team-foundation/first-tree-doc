# Quick Start

The fastest way to start a Context Tree is to let your coding agent do it.

## Option 1: Give this to your coding agent

Open Claude Code, Cursor, or any coding agent and paste this:

```
Initialize a Context Tree for my team.

Read the Context Tree spec at:
https://github.com/agent-team-foundation/context-tree-doc

Then create a tree for my project with:
- A root NODE.md describing our organization
- Domain folders based on our project structure
- A members/ folder with my personal node
- An AGENT.md with before/during/after task instructions

Organize domains by concern (what an agent needs to know to act),
not by repo or team structure. Every folder must have a NODE.md
with title and owners in the frontmatter.
```

The agent will read the spec, understand your project, and create the tree structure for you. It will ask you questions about your team, your domains, and your ownership model.

This is the intended workflow. The tree is designed to be created and maintained by agents.

## Option 2: Fork first-tree

If you want to start from the reference implementation:

```
git clone https://github.com/agent-team-foundation/first-tree.git my-tree
cd my-tree
```

Then give it to your coding agent:

```
I just cloned first-tree as a starting point.
Read NODE.md and the spec files, then adapt this tree
for my project. Replace the example content with our
actual domains, members, and decisions.
```

## Option 3: Start from scratch manually

If you prefer to do it by hand, create a repo with this minimal structure:

```
my-tree/
├── NODE.md          ← root: what is this org
├── members/
│   └── NODE.md      ← who is on the team
└── AGENT.md         ← instructions for agents
```

The root NODE.md needs:

```yaml
---
title: "My Team"
owners: [your-github-username]
---

# My Team

What this organization does and why.
```

Then grow from there. Add domain folders as you make decisions. Each folder gets a NODE.md. See [03.04 Growing Your Tree](03.04-growing-your-tree.md) for when to split and restructure.

## After setup

Once your tree exists:

1. **Push it.** The tree is a git repo. Push to GitHub. Set up CODEOWNERS from ownership declarations.
2. **Point your agents at it.** Configure your coding agent to read the tree before every task. The AGENT.md tells them how.
3. **Start recording decisions.** Every design decision, every "why we chose X over Y," goes in a leaf node. The tree grows with your team.
4. **Let agents maintain it.** After every task, the agent asks: does the tree need updating? If yes, it opens a PR. The node owner reviews.
