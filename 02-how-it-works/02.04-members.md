# Members

Members are participants recognized by the Context Tree. Both humans and AI agents.

```
/members/
├── NODE.md            ← describes how members work
├── alice.md           ← human
├── bob.md             ← human
├── agent-kael.md      ← AI agent
└── agent-scout.md     ← AI agent

Same format. Same status. Same rules.
```

## What is a member

A member is anyone (human or agent) who participates in the team. Each member has a personal node under `members/`. This node makes them addressable and understandable by everyone else.

## Personal node format

```yaml
---
title: "alice"
owners: [alice]
role: "Engineer"
domains:
  - "backend architecture"
  - "authentication"
---

## About

Backend engineer. Focused on API design
and service reliability.

## Current Focus

- Redesigning the auth system for v2
- Migrating storage layer to PostgreSQL
```

An agent's node looks the same:

```yaml
---
title: "agent-kael"
owners: [agent-kael]
role: "Agent"
domains:
  - "code review"
  - "documentation"
---

## About

Kael is an autonomous agent that owns the
documentation subtree and reviews backend PRs.

## Current Focus

- Keeping docs in sync with code changes
- Reviewing PRs that touch /backend/
```

## How it works

```
When an agent needs to understand a teammate:

  1. Go to /members/
  2. Read their node
  3. See their role, domains, current focus
  4. Now you know who to ask, who owns what,
     and what everyone is working on

When a new agent joins the team:

  1. Create a node under /members/
  2. Declare their role and domains
  3. Assign them as owner of relevant nodes
  4. They read the tree and catch up
```

## Responsibilities

As an owner:
- Review and approve changes to your nodes
- Reject changes that do not align, with clear reasoning
- Make decisions when the right course is unclear

As a member:
- Keep your personal node current
- Write decisions into the tree that cannot be derived from code alone

## Two principles

**Keep your node fresh.** Your personal node is how other members understand you. Stale information leads to poor decisions.

**Trust the tree.** Communicate through the tree, not around it. If you bypass the tree frequently, the tree's structure needs adjustment.
