# Frontmatter Reference

Every node in a Context Tree can have YAML frontmatter at the top of the file. This is the machine-readable metadata that agents and tools use.

## Format

```yaml
---
title: "Backend Architecture"
owners: [alice, bob]
soft_links: [/frontend/api-contract.md]
status: approved
role: "Engineer"
domains:
  - "backend"
  - "infrastructure"
---
```

## Fields

### title

The display name of the node.

```yaml
title: "Authentication"
```

Required on all nodes. Used by agents and tools to identify the node without reading the full content.

### owners

A list of people or agents who can approve changes to this node.

```yaml
owners: [alice, bob]
owners: [agent-kael]
owners: [*]                # anyone can approve
owners: []                 # inherit from parent NODE.md
```

Required on NODE.md files. Optional on leaf nodes (inherits from parent NODE.md if not declared). `owners: []` explicitly means "inherit from the nearest parent that declares owners." See [02.03 Ownership](02.03-ownership.md) for the six rules.

> **Note:** GitHub's YAML parser requires the wildcard to be quoted: `owners: ["*"]`. The spec uses `owners: [*]`.

### soft_links

A list of paths to related nodes in other parts of the tree.

```yaml
soft_links: [/backend/auth.md, /infrastructure/identity.md]
```

Optional. See [02.05 Soft Links](02.05-soft-links.md).

### status

The state of a decision or proposal.

```yaml
status: draft
status: approved
status: deprecated
```

Optional. Useful for decision nodes and proposals.

### role

The role of a member. Used in personal nodes under `members/`.

```yaml
role: "Engineer"
role: "Agent"
role: "Product Lead"
```

Optional. Only relevant for member nodes.

### domains

The broad areas of responsibility for a member.

```yaml
domains:
  - "backend architecture"
  - "authentication"
```

Optional. Only relevant for member nodes. Describes areas of focus, not specific node paths. Concrete ownership is determined by the `owners` field in each node.

## Example: NODE.md

```yaml
---
title: "Backend"
owners: [alice]
soft_links: [/infrastructure/deployments.md]
---

This subtree covers backend architecture,
API design, and data access patterns.
```

## Example: leaf node

```yaml
---
title: "Authentication"
owners: [carol]
status: approved
soft_links: [/frontend/login.md]
---

We use JWT tokens with refresh rotation.
```

## Example: member node

```yaml
---
title: "alice"
owners: [alice]
role: "Engineer"
domains:
  - "backend architecture"
  - "authentication"
---

## About

Backend engineer focused on API design.

## Current Focus

- Auth system v2 redesign
```
