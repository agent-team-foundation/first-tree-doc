# The Tree

## What is a Context Tree

A Context Tree is a Git repository where folders are subtrees and markdown files are nodes.

That is the whole thing. Folders and files. No database. No special tooling. Just a git repo.

```
my-org-tree/
├── NODE.md              ← root node
├── backend/
│   ├── NODE.md
│   ├── auth.md
│   └── storage.md
├── frontend/
│   ├── NODE.md
│   └── design.md
└── members/
    ├── NODE.md
    ├── alice.md
    └── agent-kael.md
```

## Three concepts

A Context Tree has three building blocks.

```
┌──────────────────────────────────────────────────┐
│                                                   │
│   1. Folders = subtrees                           │
│      A folder represents a domain.                │
│      /backend/ is the backend domain.             │
│      /members/ is where the team lives.           │
│                                                   │
│   2. NODE.md = the folder's identity              │
│      Every folder MUST have one.                  │
│      It describes the domain and declares         │
│      who owns it.                                 │
│                                                   │
│   3. Leaf files = the content                     │
│      Any .md file that is not NODE.md.            │
│      auth.md, storage.md, alice.md.               │
│      This is where decisions and context live.    │
│                                                   │
└──────────────────────────────────────────────────┘
```

## How agents navigate it

An agent does not search the tree. It walks the tree.

```
Agent starts at:   /members/agent-kael.md    (home node)
                          │
Walks up to:       /members/NODE.md          (who else is on the team?)
                          │
Walks up to:       /NODE.md                  (what is this org about?)
                          │
Walks down to:     /backend/NODE.md          (what does backend cover?)
                          │
Walks down to:     /backend/auth.md          (what decisions were made?)
                          │
Now acts:          with full context.
```

Start at your node. Walk up for broader context. Walk down for detail. Read NODE.md at each level to orient yourself.

The path IS the address. `/backend/auth.md` tells you exactly where the node is and what it is about.

## Why git

The tree is a git repository. This gives you:

- **History.** Every change is a commit. You see who wrote what and when.
- **Review.** Pull requests work out of the box. Changes go through approval.
- **Enforcement.** CODEOWNERS can be generated from the tree's ownership.
- **No lock-in.** Markdown files in a git repo. Host it anywhere.
- **Compose.** Works with existing developer workflows. No new tools required.
