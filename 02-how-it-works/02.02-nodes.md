# Nodes

There are two types of nodes in a Context Tree.

```
┌─────────────────────────┐     ┌─────────────────────────┐
│       NODE.md            │     │      Leaf Node          │
│                          │     │                         │
│  One per folder.         │     │  Any other .md file.    │
│  Describes the domain.   │     │  Contains the content.  │
│  Declares the owner.     │     │  Decisions, designs,    │
│  The folder's identity.  │     │  context.               │
└─────────────────────────┘     └─────────────────────────┘
```

## NODE.md

Every folder in the tree **must** contain a `NODE.md` file (uppercase). It serves two purposes:

1. **Description.** The human and agent readable explanation of what this folder covers.
2. **Ownership.** The frontmatter declares who owns all content within the folder.

Here is what a NODE.md looks like:

```yaml
---
title: Backend Architecture
owners: [alice, bob]
soft_links: [/infrastructure/deployments]
---

This subtree covers the backend service architecture,
including API design, data access patterns, and
service-to-service communication.
```

When an agent arrives at a folder, the first thing it reads is NODE.md. This tells it: what is this domain, who owns it, and what else is related.

## Leaf nodes

Any `.md` file in a folder other than `NODE.md` is a leaf node. This is where the actual content lives.

```
/backend/
  NODE.md          ← "what is the backend domain?"
  auth.md          ← decisions about authentication
  storage.md       ← decisions about data storage
  api-v2.md        ← the v2 API redesign and why
```

Leaf nodes can have their own frontmatter:

```yaml
---
title: Authentication
owners: [carol]
soft_links: [/frontend/login.md]
---

We use JWT tokens with refresh rotation.
The previous session-based approach was
abandoned in Q3 because...
```

Leaf node owners are **additive**. If `NODE.md` says `owners: [alice]` and `auth.md` says `owners: [carol]`, then both Alice and Carol can approve changes to `auth.md`.

## The rule

```
NODE.md exists?
  │
  ├── YES → valid folder
  │
  └── NO  → not a valid Context Tree folder
              (every folder must have one)
```

This is enforced. You cannot have a folder without a NODE.md. Creating a subfolder means someone has to think about what this domain is and who owns it. This prevents folder sprawl.
