# The Problem

## How teams use agents today

You open a session with an AI agent. It has no idea what your team has been building. So you explain.

Here is our stack. Here is why we chose a monolith. Here is who owns what. Here is what we tried last quarter.

The agent gives you something. It is decent but it misses context. You correct. You re-prompt. You paste in docs. Thirty minutes later, you have something usable.

Now you need a second agent for the frontend. That agent also knows nothing. You explain again.

```
              ┌─────────┐
   context    │         │    context
 ┌───────────▶│  Human  │◀────────────┐
 │            │         │             │
 │            └────┬────┘             │
 │                 │                  │
 │        "let me explain             │
 │         everything again..."       │
 │                 │                  │
 ▼                 ▼                  ▼
┌────────┐   ┌────────┐        ┌────────┐
│Agent A │   │Agent B │        │Agent C │
│        │   │        │        │        │
│no      │   │no      │        │no      │
│memory  │   │memory  │        │memory  │
└────────┘   └────────┘        └────────┘

Each agent is an island.
The human is the only bridge.
```

We call this _context routing_: the human manually ferrying knowledge between agents that cannot see each other.

You added three agents to your team. You got three times the context-routing work.

## Why this happens

The bottleneck is not intelligence. These models can write code, reason about architecture, debug production issues. They are genuinely good.

The bottleneck is _memory_.

Think about what makes a senior engineer effective.

```
┌──────────────────────┐     ┌──────────────────────┐
│   Senior Engineer    │     │      AI Agent         │
│                      │     │                       │
│ ✓ knows the history  │     │ ✗ starts from zero    │
│ ✓ knows the owners   │     │ ✗ no idea who owns    │
│ ✓ knows what failed  │     │   what                │
│ ✓ knows the why      │     │ ✗ no idea what was    │
│                      │     │   tried before        │
│ = fast, aligned      │     │ = capable but lost    │
└──────────────────────┘     └──────────────────────┘

The difference is not skill. It is context.
```

A senior engineer is not 10x smarter than a junior. They carry _context_. They know which approaches were tried and abandoned. They know why things are the way they are. That accumulated context is what makes them fast.

AI agents have none of this. Every session is a blank slate.

You can give them a `CLAUDE.md` or a system prompt. But that is a flat document. It does not capture the relationships between decisions. It does not update itself. It does not scale.

## What is missing

```
Team knowledge today:

  Meetings → no trace
  Slack threads → buried
  Docs → out of date
  PRs → merged and forgotten
  Someone's head → they left

  ┌─────────────────────────────┐
  │  The reasoning disappears.  │
  │  The output remains.        │
  │  But the WHY is gone.       │
  └─────────────────────────────┘
```

The gap between a great human teammate and an AI agent is not intelligence.

It is context.

Context Tree closes that gap.
