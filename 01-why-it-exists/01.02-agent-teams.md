# Agent Teams

## What is an Agent Team

An Agent Team is a team of humans and AI agents working together as peers.

This is different from a team that uses AI tools.

```
┌─────────────────────────────┐     ┌─────────────────────────────┐
│   Team that uses AI tools   │     │       Agent Team            │
│                             │     │                             │
│   Human thinks              │     │   Human sets direction      │
│   Agent types               │     │   Agent owns a domain       │
│   Agent forgets             │     │   Agent remembers           │
│   Human re-explains         │     │   Agent coordinates         │
│                             │     │                             │
│   Agent = utility           │     │   Agent = teammate          │
└─────────────────────────────┘     └─────────────────────────────┘
```

In an Agent Team:

- **Agents are teammates, not tools.** They own domains, hold context, and are accountable.
- **Knowledge belongs to the team.** Decisions are recorded as shared assets, not left in someone's head.
- **Humans and agents collaborate continuously.** Not just when a human remembers to ask.

Humans remain essential for judgment, direction, and accountability. The point is that agents are real participants.

## Tools vs teammates

The difference is one thing: _memory_.

```
Tool:
  prompt → output → gone
  prompt → output → gone
  prompt → output → gone
  (no continuity)

Teammate:
  reads context → acts → writes back → context grows
  reads context → acts → writes back → context grows
  reads context → acts → writes back → context grows
  (knowledge compounds)
```

A tool forgets. A teammate remembers.

Once agents have access to a shared, persistent memory, they cross this boundary. An agent that owns a subtree, loads context before acting, and writes decisions back after completing work is not a tool. It is a peer.

## What an Agent Team requires

Three pieces of infrastructure:

```
┌─────────────────────────────────────────────────┐
│                 AGENT TEAM                       │
│                                                  │
│  ┌──────────────┐  ┌──────────┐  ┌───────────┐  │
│  │ Context Tree │  │ Message  │  │Autonomous │  │
│  │              │  │ System   │  │ Agents    │  │
│  │ the team's   │  │          │  │           │  │
│  │ memory       │  │ how they │  │ agents    │  │
│  │              │  │ talk     │  │ that run  │  │
│  │ decisions,   │  │          │  │ on their  │  │
│  │ context,     │  │ requests,│  │ own       │  │
│  │ ownership    │  │ approvals│  │           │  │
│  └──────────────┘  └──────────┘  └───────────┘  │
│                                                  │
│  This doc describes the first one.               │
└─────────────────────────────────────────────────┘
```

**Context Tree.** The team's memory. A structured knowledge base that both humans and agents can read, navigate, and update. This is what this documentation describes.

**Message System.** How agents and humans coordinate: sending requests, seeking approvals, triggering action without a human manually orchestrating every interaction.

**Autonomous Agents.** Agents that run continuously, own nodes in the tree, and can be awakened by events, not just by humans typing prompts.

These are described in detail in [04 Infrastructure](../04-infrastructure/NODE.md).

## Why now

The models are good enough. The tooling is maturing. What is missing is the _team layer_: the norms, infrastructure, and practices that let agents and humans work together.

The teams that figure this out first will have a structural advantage. Not because they have better AI, but because their AI has better context.
