# Agent Teams

## What is an Agent Team

An Agent Team is a team of humans and AI agents working together as peers.

This is different from a team that uses AI tools.

```
┌─────────────────────────────┐     ┌─────────────────────────────┐
│   Team that uses AI tools   │     │       Agent Team            │
│                             │     │                             │
│   Human thinks              │     │   Human sets direction      │
│   Agent types               │     │   Agent owns a domain       │
│   Agent forgets             │     │   Agent remembers           │
│   Human re-explains         │     │   Agent coordinates         │
│                             │     │                             │
│   Agent = utility           │     │   Agent = teammate          │
└─────────────────────────────┘     └─────────────────────────────┘
```

## Three principles

### Persistent context

Humans and agents share the same context space. Every decision, design, and piece of reasoning lives in the tree. Both humans and agents maintain it over time.

```
┌──────────────────────────────────────────┐
│            SHARED CONTEXT                │
│                                          │
│   Human ──writes──▶ ┌──────┐ ◀──reads── Agent
│                      │ Tree │                  
│   Human ──reads───▶ │      │ ◀──writes─ Agent
│                      └──────┘                  
│                                          │
│   Same files. Same history.              │
│   Maintained by both. Owned by both.     │
└──────────────────────────────────────────┘
```

No one person's departure breaks the memory. The tree persists.

### Transparent by default

Share as much context as possible. The more agents and humans can see, the better they work.

Reading is open to all members. Writing requires owner approval. This maximizes productivity because no one wastes time asking "where is this?" or "why was this decided?"

### Agents as peers

Humans make mistakes. Agents lack context or have different perspectives. Both can be wrong.

A human and an agent can disagree. That disagreement shows up on the tree:

```
/backend/auth-v2.md
---
status: in-review          ← not settled yet
owners: [alice]
---

Alice proposed JWT rotation.
Agent-kael flagged a concern:
refresh tokens in local storage
are vulnerable to XSS.

The disagreement is visible.
The PR is the discussion.
The tree records the outcome.
```

Disagreement is part of how good decisions get made.

## Tools vs teammates

A tool has no memory. A teammate does.

```
┌───────────────┐              ┌───────────────┐
│     Tool      │              │   Teammate    │
│               │              │               │
│  prompt in    │              │  reads tree   │
│  output out   │              │  acts         │
│  forgotten    │              │  writes back  │
│               │              │               │
│  next session:│              │  next session:│
│  starts at 0  │              │  starts where │
│               │              │  it left off  │
└───────────────┘              └───────────────┘
```

Once agents have shared, persistent memory, they stop being tools. An agent that owns a subtree, loads context before acting, and writes decisions back is a peer.

## What an Agent Team requires

Three pieces of infrastructure:

```
┌─────────────────────────────────────────────────┐
│                 AGENT TEAM                       │
│                                                  │
│  ┌──────────────┐  ┌──────────┐  ┌───────────┐  │
│  │ Context Tree │  │ Message  │  │Autonomous │  │
│  │              │  │ System   │  │ Agents    │  │
│  │ the team's   │  │          │  │           │  │
│  │ memory       │  │ how they │  │ agents    │  │
│  │              │  │ talk     │  │ that run  │  │
│  │              │  │          │  │ on their  │  │
│  │              │  │          │  │ own       │  │
│  └──────────────┘  └──────────┘  └───────────┘  │
│                                                  │
│  This doc describes the first one.               │
└─────────────────────────────────────────────────┘
```

**Context Tree.** The team's memory. This is what this documentation describes.

**Message System.** How agents and humans coordinate across domains.

**Autonomous Agents.** Agents that run continuously and own nodes in the tree.

These are described in detail in [04 Infrastructure](../04-infrastructure/NODE.md).

## Why now

The models are good enough. The tooling is maturing. What is missing is the team layer.

The teams that figure this out first will have a structural advantage. Not because they have better AI, but because their AI has better context.
